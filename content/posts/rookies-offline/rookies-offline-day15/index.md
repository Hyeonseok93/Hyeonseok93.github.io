## ① A팀 — 진단 엔진이 커버할 체크리스트

A팀은 DAST(ZAP)·SAST(Semgrep)와 AI 우선순위 모듈을 맡습니다. 엔진이 겨냥하는 진단 범위는 아래 체크리스트입니다. *(일부 항목은 동적 자동 진단이 어렵고 수동·설정 점검 영역으로 둡니다.)*

### 1. 입출력 값 검증 부재

- **XSS / CSRF (1-1)**
  - *Reflected XSS:* 검색창·쿼리 파라미터가 응답에 즉시 스크립트로 렌더링
  - *Stored XSS:* 게시글·댓글 등 DB에 저장된 스크립트가 조회 시 실행
  - *DOM-based XSS:* `innerHTML`, `eval` 등 클라이언트 DOM 조작 취약 흐름
  - *CSRF:* Anti-CSRF 토큰 누락으로 로그인 상태에서 원치 않는 변경·결제 API 호출 유도
- **Injection — SQL / Command / XXE / XPATH / LDAP (1-2)**
  - *SQL Injection:* Boolean·Error·Union·Time-based 등으로 쿼리 구조 변조
  - *OS Command Injection:* `;` `&` `|` 등으로 웹 서버 권한 OS 명령 실행
  - *XXE:* 외부 엔티티로 로컬 파일 조회·내부 포트 스캔
  - *XPATH / LDAP / SSI:* 검색·인증 쿼리 구조 조작으로 우회·디렉터리 탈취
- **파라미터·히든 필드 조작 (1-3)**
  - *수평 파라미터 조작:* `amount` 등 결제·비즈니스 값 위조
  - *히든 필드 우회:* `role=user` 등 hidden 값 변조로 서버 검증 우회
- **SSRF / File Inclusion (1-4)**
  - *SSRF:* 로고 URL 등으로 내부망(`localhost` 관리 포트 등) 요청 유도
  - *LFI:* `../` 경로 순회로 `/etc/passwd` 등 로드
  - *RFI:* 외부 악성 스크립트 URL 동적 로드·실행
- **검증되지 않은 리다이렉트·포워드 / CORS (1-5)**
  - *Open Redirect:* `nextUrl` 등으로 피싱 도메인 강제 이동
  - *CORS 오설정:* `Allow-Origin: *` + `Allow-Credentials: true` 조합으로 자격증명 탈취 여지
- **입력 크기·무결성 — 역직렬화 등 (1-6)**
  - *안전하지 않은 역직렬화:* 신뢰할 수 없는 스트림을 `ObjectInputStream` 등으로 복원하며 가젯 체인·RCE

### 2. 취약한 파일 처리

- **악성 파일 업로드 (2-1):** 웹셸(JSP/PHP 등), 대소문자·이중 확장자·널바이트 등 확장자 필터 우회
- **중요 정보 파일 다운로드 (2-2):** 다운로드 파라미터에 `../` Path Traversal로 설정·소스 무단 취득

### 3. 취약한 접근 통제

- **패스워드 정책 (3-1, 동적 진단 곤란):** 복잡도 미흡, HTTPS/암호화 없이 평문 전송
- **인증 실패 횟수 제한 (3-2, 동적 진단 곤란):** Brute-force에 잠금·Backoff·CAPTCHA 부재
- **계정 정보 파악 (3-3):** 댓글·게시 API DTO에 이메일·권한 평문, 마스킹 누락
- **관리자 페이지 분리 (3-4):** `/admin` 등 추측 가능 경로 노출, `admin/admin` 디폴트 계정
- **검색엔진 정보 노출 (3-5):** `robots.txt` Disallow 누락으로 관리 경로 인덱싱
- **백업·테스트 파일 (3-6):** `.bak` `.zip` `db_dump.sql`, 프로덕션에 남은 TestController 등

### 4. 취약한 인증·세션

- **쿠키·웹 스토리지 (4-1):** HttpOnly/Secure 누락, 로컬스토리지에 토큰·권한 평문 저장
- **세션·토큰 안전성 (4-2, 일부 동적 진단 곤란):** 로그아웃 시 Redis Refresh 미파기, 로그인 전후 세션 고정
- **접근제어 우회·IDOR (4-3):** 예약 `id` 등 타인 리소스 강제 조회·수정
- **비인증 중요 페이지 (4-4):** Security 허용 목록 오설정으로 결제 등 API 비로그인 호출
- **일반계정 권한 상승 (4-5):** `role=admin` 파라미터로 수직 권한 상승

### 5. 중요 정보 저장·전송

- **소스 내 주요정보 (5-1, 동적 진단 곤란):** API Key·DB 암호·JWT Secret 하드코딩 (Semgrep 쪽)
- **요청·응답 내 주요정보 (5-2):** 주민·계좌·카드 등 평문 노출 (Regex·응답 검사)

### 6. 부적절한 오류 처리

- **Stack Trace 유출 (6-1):** 500 응답에 클래스·경로·버전 노출
- **일괄 오류·로그인 메시지 (6-2):** Whitelabel Error Page, ID 존재 여부를 가르는 로그인 실패 메시지

### 7. 세션 통제·환경 설정

- **HTTP Method 제한 (7-1):** PUT/DELETE/TRACE 등 불필요 메서드 개방
- **디렉터리 리스팅 (7-2):** `/images/` 등 index 없이 파일 목록 노출
- **서버 헤더 노출 (7-3):** `Server`, `X-Powered-By` 버전 배너
- **취약한 보안 설정 (7-4):** 알려진 CVE 종속성, TrustManager 우회로 SSL 검증 비활성

### 8. 기타

- **가이드 외·비즈니스 로직 (8-1, 수동 영역):** Race Condition, 포인트 이중 사용 등 Zero-Day·설계 결함 — 자동 DAST 범위 밖

## ③ C팀 — 한글 PDF 리포트 · API · AWS 배포

C팀 목표는 A팀 분석 데이터와 B팀 증적 이미지를 합쳐 **KISA 주요정보통신기반시설(주통기) 가이드라인에 매핑되는 한글 PDF**를 만들고, **ECS Fargate + CloudFront**로 서비스에 올리는 것입니다.

| 구분 | 역할 A — PDF·규제 매핑·ECS | 역할 B — API·템플릿·CloudFront |
| --- | --- | --- |
| **핵심** | WeasyPrint, 한글 폰트/레이아웃, KISA 매핑, ECS Fargate | FastAPI·Pydantic, 무결성 검증, Jinja2 HTML, CloudFront·HTTPS |

### PDF·ECS 쪽 태스크

- **WeasyPrint 환경:** Docker에 WeasyPrint·리눅스 종속성 고정, API와 분리된 워커
- **한글·인쇄 레이아웃:** `Noto Sans KR` 임베드, paged media로 여백·헤더·푸터·페이지 번호
- **변환 API:** `POST /api/v1/reports/generate` 비동기 처리 → PDF를 S3 업로드 → Presigned URL 반환
- **KISA 매핑:** ZAP·Semgrep raw를 주통기 표준 항목 코드로 분류하는 매핑 테이블·스크립트
- **ECS Fargate:** PDF 렌더링용 태스크 정의·자원 분리

### API·템플릿·CDN 쪽 태스크

- **FastAPI·Pydantic:** A팀 취약점·B팀 이미지 URL 스키마, 타입·필수값 검증
- **무결성·예외:** 필드 누락 처리, S3 URL 접근성 검사, 이미지 없을 때 대체 텍스트
- **기술팀 HTML:** CVE·재현 경로·스크린샷용 다크 테마, 위험도별 색상 (Jinja2)
- **경영진 요약 HTML:** 등급 차트·준수율 점수, AI ‘이번 달 조치’ 슬롯
- **CloudFront:** S3 앞단 CDN, 캐시·ACM 커스텀 도메인·HTTPS
