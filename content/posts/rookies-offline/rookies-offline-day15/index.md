# 1. 아르고스(Argus) 명칭 확정 및 독립 Git 체계

14일차에 잡은 웹 취약점 자동 진단 플랫폼 기획을 이어, 이름을 확정하고 온데와 저장소를 분리했습니다.

- **플랫폼 명칭:** 팀 논의를 거쳐 **'아르고스(Argus)'**로 확정했습니다. 그리스 신화의 아르고스처럼 빈틈없이 취약점을 잡겠다는 의미를 담았습니다.
- **전용 GitHub 오거나이제이션:** 진단 대상인 '온데(onde)' 저장소와 섞이지 않도록 전용 오거나이제이션(**`https://github.com/UR-ARGUS`**)과 공식 레포를 새로 만들었습니다.

# 2. A / B / C 팀 분담

엔진·증적 캡처·리포트 쪽을 병렬로 가져가기 위해 작업을 A/B/C 세 팀으로 나눴습니다.

- **A팀 — 진단 엔진·AI:** DAST(ZAP)·SAST(Semgrep) 연동과, AI로 취약점 우선순위를 나누는 쪽을 맡습니다.
- **B팀 — Selenium·증적 캡처:** EC2/S3 환경에서 헤드리스 브라우저로 진단 순간을 캡처하고, 이미지를 S3에 올리는 쪽을 맡습니다.
- **C팀 — 리포트·서비스 통합:** 스캔 결과와 캡처 메타데이터를 받아 한글 PDF/HTML로 내고, API 계약·데이터 무결성·최종 배포를 맞춥니다.

# 3. 5대 필수 설계 산출물

마일스톤에 맞춰 **5대 설계 산출물**과 진단 시나리오 문서를 작성해 구글 드라이브에 올렸습니다.

1. **프로젝트 수행 계획서:** 타임라인, 리스크, 딜리버리 로드맵
2. **요구사항 정의서:** 동적/정적 스캔, AI 분석 등 충족해야 할 사양
3. **백엔드 API 명세서:** 프론트–백엔드 인터페이스와 상태 조회 엔드포인트
4. **데이터베이스 도메인 명세서:** 취약점 히스토리·세션 매핑용 ERD/스키마
5. **핵심 컴포넌트 기능 정의서:** 엔진 연동 스케줄러, 파일 파서 등 서브 모듈 동작
