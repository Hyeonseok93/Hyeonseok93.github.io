# 서론

> **"차세대 보안 진단 플랫폼 '아르고스(Argus)'의 공식 아이덴티티를 확정하고, 독립 Git 형상관리 체계·3대 전담 팀 R&R·5대 필수 설계 산출물·AWS 실서버 및 S3 증적 스토리지 연동까지 한 번에 가동했습니다."**
>
> 아르고스(Argus) 공식 명칭과 전용 GitHub 오거나이제이션을 확정하고, 3대 전담 팀 R&R과 5대 필수 설계 산출물 작성을 완료했습니다.

# 1. 차세대 보안 진단 플랫폼 명칭 확정 및 독립 형상관리(Git) 체계 수립

14~15일차에 진행된 범용 웹 보안 진단 플랫폼의 기획 고도화에 이어, 2026년 6월 10일에는 우리 플랫폼의 공식 아이덴티티를 확정하고 독립적인 개발 인프라를 가동했습니다.

- **플랫폼 공식 명칭 확정:** 팀 전원의 심도 있는 논의를 거쳐 차세대 보안 진단 시스템의 이름을 **'아르고스(Argus)'**로 최종 확정했습니다. 그리스 신화 속 100개의 눈을 가정한 채 잠들 때조차 상시 감시를 멈추지 않는 거인처럼, **진단 대상 웹 서비스의 아주 미세한 보안 취약점 결함까지 빈틈없이 탐지해 내겠다**는 플랫폼의 강력한 지향점을 투영했습니다.
- **전용 GitHub 오거나이제이션 개설:** 진단 대상인 여행 플랫폼 '온데(onde)' 저장소와 소스코드가 뒤섞이는 현상을 방지하고 완전한 범용 스캐너 소프트웨어로서의 형상 관리를 수행하기 위해, 전용 GitHub 오거나이제이션(**`https://github.com/UR-ARGUS`**) 및 공식 레포지토리를 신규 개설했습니다 (유지수 팀원 담당).

# 2. 전문화 협업을 위한 진단 플랫폼 3대 전담 팀 구성 및 R&R 분담

이기종 스캔 엔진 제어, 공간 증적 자원 캡처, 지능형 데이터 분류라는 복합 아키텍처를 효율적으로 병렬 빌드하기 위해, 플랫폼 영역을 세분화하여 **3개의 전담 전문화 조직으로 R&R을 정밀 분담**했습니다.

- **A팀 [진단 엔진 및 AI 코어]:** DAST(ZAP) 및 SAST(Semgrep) 연동 파이프라인의 설계와 AI API 프롬프트 엔지니어링을 통한 취약점 우선순위 분류 서브 모듈 프로토타입 조사를 집중 수행합니다.
- **B팀 [Selenium 스크린 자동 캡처 및 스캐너]:** 윈도우 서버(Windows Server) 환경 하에 헤드리스 브라우저를 안정적으로 구동하고, 진단 순간의 스크린샷을 자동 자산화하기 위한 Selenium 윈도우 크롤러 스크립트 세부 리서치 및 연동을 전담합니다.
- **C팀 [리포트 자동 생성 및 서비스 통합]:** 스캔 통계 로우 데이터와 AI 보완문을 파싱하여 고품질 한글 보고서(PDF)로 자동 출력하는 엔진 설계 및 전반적인 진단 웹 서비스 통합 구조 설계를 담당합니다.

# 3. 플랫폼 엔지니어링의 뼈대: 5대 필수 설계 산출물 및 시나리오 작업 완수

체계적인 개발 거버넌스 준수와 철저한 요구사항 추적성을 확보하기 위해, 마일스톤 가이드라인에 명시된 **5대 필수 설계 산출물과 범용 웹 모의해킹 관점의 진단 시나리오 문서 작업**을 100% 완료하고 구글 드라이브에 공식 업로드 공유했습니다.

1. **플랫폼 프로젝트 수행 계획서:** 플랫폼 아르고스의 개발 타임라인, 리스크 관리, 최종 스펙 딜리버리 로드맵에 대한 원본 및 PDF 산출물 빌드를 마쳤습니다.
2. **플랫폼 요구사항 정의서:** 동적/정적 스캔 요청 기능, AI 분석 기능 등 시스템이 충족해야 하는 상용 사양 요소를 정밀 명세화했습니다.
3. **플랫폼 백엔드 API 명세서:** 프론트-백엔드 간 인터페이스 통신 규격과 실시간 상태 인덱싱을 위한 전용 엔드포인트 규칙 수립을 완수했습니다.
4. **플랫폼 데이터베이스 도메인 명세서:** 취약점 히스토리 관리 및 세션 매핑을 위한 논리/물리 ERD 데이터 모델링 및 스키마 설계를 확정했습니다.
5. **플랫폼 핵심 컴포넌트 기능 정의서:** 이기종 엔진 연동 스케줄러, 파일 파서 등 핵심 서브 모듈 컴포넌트들의 세부 동작 사양 명세를 작성했습니다.

# 4. 인프라 트랙 성과: 백엔드 실서버 배포 및 AWS S3 스토리지 결합 완료

인프라 전담팀(유지수, 박진아)의 기민한 아키텍처 대응을 통해 아르고스 플랫폼의 백엔드 시스템 구동 서버 환경과 증적 저장을 위한 클라우드 스토리지 자원 연동을 선제적으로 완비했습니다.

- **클라우드 실서버 환경 배포 완수:** 아르고스 백엔드 코어 시스템을 AWS 클라우드 네트워크 환경 인프라에 프로비저닝 완료하여 실서버 구동성을 확보했습니다.
- **증적 자원 AWS S3 버킷 구축:** 진단 자동화 프로세스 중 Selenium 크롤러 엔진이 캡처해내는 실시간 웹 취약점 증적 이미지 파일들을 안전하고 고속으로 휘발 없이 저장할 수 있도록 **보안 오브젝트 스토리지 AWS S3 버킷을 신설하고 백엔드 애플리케이션과의 API 연동 체결을 완료**했습니다.

# 5. Next Step: 기획 정교화 클로징 및 자체 플랫폼 대상 실전 취약점 진단 테스트 착수

설계 아티팩트와 배포 인프라 셋업이 무결하게 완비됨에 따라, 우리 팀은 다가오는 6월 11일(목) 스프린트를 기점으로 서류 단계를 넘어 실제 '수동 진단 및 멘토링 대비 피드백 페이즈'로 직진합니다.

- **주요 마일스톤 계획:** 목요일 오전 타임라인을 활용해 기획 세부 고도화 및 시스템 아키텍처 리서치를 최종 1차 클로징하고 멘토링 전 문서 작성을 정돈합니다. 오후 시간부터는 SK쉴더스 컴플라이언스 가이드라인을 기반으로 우리가 선제 구축해 둔 **여행 플랫폼 '온데(onde)' 시스템에 Burp Suite 등 수동 진단 도구를 직접 투입하여 자체 플랫폼 모의 웹 취약점 진단 실전 테스트에 착수**합니다.
