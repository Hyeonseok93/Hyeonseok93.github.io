# 서론

> **"진단의 대상이 되는 여행 플랫폼 '온데(onde)'의 최종 결합도와 데이터 무결성을 위한 스키마 수정을 마무리했습니다. 이와 동시에, '공격을 알아야 방어가 가능하다'는 핵심 기치 아래 온데를 정밀 스캔할 '웹 취약점 자동 진단 플랫폼'의 구조적 기획을 수립했습니다. 오픈소스(ZAP·Semgrep) 분석을 바탕으로 AI 우선순위 분류 및 Selenium 자동 증적 캡처를 포함한 3대 핵심 MVP 기능을 확정하며 독보적인 보안 프로젝트의 이원화 아키텍처를 구현했습니다."**
>
> 온데(onde)를 진단 대상으로 두고 ZAP·Semgrep·SSL Labs 기반 범용 웹 취약점 자동 진단 플랫폼의 3대 MVP 기능을 확정했습니다.

# 1. 진단 대상 '온데(onde)' 여행 플랫폼 코어 고도화 및 스키마 대수술

본격적인 취약점 스캐닝과 모의해킹의 타깃(Target)이 될 여행 플랫폼 '온데(onde)'의 안정성을 프로덕션 레벨로 격상하고, 프론트엔드와 백엔드 간의 최종 결합 상태를 점검하며 데이터 계층 리팩토링을 완수했습니다.

- **도메인 핵심 스키마 추가 및 확장:**
  - 판매자 정산 및 계정 관리 무결성을 위해 `seller_account` 구조를 `V1` 테이블 초기 형성에 최종 추가 반영했습니다.
  - 숙소 상세 옵션 다각화를 반영하고자 `rooms` 테이블 스펙에 신규 비즈니스 컬럼을 확장하는 `V5` 마이그레이션 스크립트를 정밀 적용했습니다.
- **Flyway 버전 파이프라인 충돌 해결:**
  - 개발팀 간 병렬 머지 과정에서 형상 관리가 꼬이며 발생한 `V9` 중복 스키마 리스크를 식별했습니다. 시스템 인스턴스 셧다운 없이 정합성을 맞추기 위해 중복 구간을 구조적으로 격리하여 **`V10` 버전으로 마이그레이션을 분리 세이빙**하는 데 성공했습니다.
- **지리 공간 데이터(LBS) 정밀화 및 마스터 데이터 교체:**
  - 지도 탐색 화면의 핀 포인트 오차를 제로화하기 위해, 데이터베이스 내에 적재되어 있던 숙소 위도 및 경도 좌표값을 전면 수정했습니다. 구글 드라이브를 통해 공유된 최신 정제 합본 CSV 데이터를 기반으로 데이터베이스 인프라 마이그레이션을 완수했습니다.
- **성능 및 기능 최적화 연동:**
  - 대량의 지리 데이터 및 최신 숙소 리스트 로드 시 브라우저 오버헤드를 방지하기 위해 `IntersectionObserver` 기반의 **숙소 카드 레이지 로딩(Lazy Loading) 아키텍처를 추가 이식**하여 렌더링 성능을 극대화했습니다.
  - 프론트엔드 포트원 결제창 요청과 백엔드 데이터베이스의 `payments/prepare` 및 `validate` 검증 계층 간의 **실결제 연동 파이프라인 최종 결합을 검증**하고, 여행기(커뮤니티) 게시글 추가/수정 폼의 데이터 바인딩 예외 처리를 깔끔하게 마감했습니다.

# 2. 웹 취약점 자동 진단 플랫폼 기획 방향성 및 정체성 수립

우리 팀 **'너의 취약점은'**은 단순한 모의해킹 훈련에 그치지 않고, 웹 서비스의 보안 결함을 자동으로 탐지·추적하는 통합 거버넌스를 구축하기 위해 **보안 진단 플랫폼의 기획 방향성과 진단 대상을 명확히 분리·확정**했습니다.

- **진단 대상과 진단 플랫폼의 명확한 이원화 구도 정립:**
  - 우리가 개발한 고도화된 여행 플랫폼 **'온데(onde)'는 취약점 진단의 대상(Target)**이 됩니다.
  - 이와 동시에, 온데의 취약점을 자동으로 스캔하고 관리자에게 시큐어 코딩 보완문을 제시하는 **'웹 취약점 자동 진단 플랫폼'의 코어 아키텍처를 상호 연계하여 함께 빌드**합니다.
- **범용 웹 진단 플랫폼으로의 확장성 확보:** 특정 도메인에 종속되는 전문 툴이 아닌, 다양한 모던 웹 서비스 환경 전체를 유연하게 스캔할 수 있는 **'범용 진단 플랫폼'**의 정체성을 수립했습니다.
- **기획 구성 비율의 황금 밸런스(50:50) 정의:** 학술적 가치와 구조적 기술 문제를 깊이 있게 다루는 **'기술 논문 형식' 50%**와, 실무 관리자의 편의성 및 가독성을 극대화하는 **'서비스 형식' 50%**의 비율을 채택하여 플랫폼의 유용성을 고조시켰습니다.
- **주제 개요 및 당위성 정의:** 왜 이 플랫폼이 필요한가에 대한 명확한 기술적 개요를 수립하고, 시나리오 기반의 취약점 탐색 프로세스를 설계함으로써 플랫폼의 존재 당위성을 확고히 다졌습니다.

# 3. 차별점 확보를 위한 오픈소스 도구 분석 및 3대 핵심 MVP 기능 도출

기존의 단순 단일 스캐너들과 확실한 기술적 차별점을 확보하기 위해 현업 표준 오픈소스 진단 도구들을 리서치하고, 이를 유기적으로 제어하는 오케스트레이션 파이프라인 및 **3대 핵심 주요 기능**을 선정하여 MVP 사양 설계를 완료했습니다.

## ① 오픈소스 진단 툴 리서치 및 라인업 확정

- **OWASP ZAP:** 동적 애플리케이션 보안 테스트(DAST) 엔진으로 채택하여 구동 중인 웹 서비스를 대상으로 실시간 패킷 퍼징 및 주입 공격 취약점 자동 스캔.
- **Semgrep:** 정적 애플리케이션 보안 테스트(SAST) 소스코드 분석 도구로 채택하여 개발 단계의 소스코드 내 시큐어 코딩 규칙 위반 여부 정밀 스캔.
- **SSL Labs:** 외부 통신망의 안전성을 보장하기 위해 웹 서버의 SSL/TLS 암호화 프로토콜 설정 및 인증서 취약점 평가.

## ② 플랫폼 3개 메인 핵심 기능 및 MVP 스펙

1. **AI 기반 조치 우선순위 분류 알고리즘:** 대량의 로우 데이터(Raw Data) 취약점이 발견되었을 때, 인공지능 API와 정교한 프롬프트 엔지니어링 설계를 통해 어떤 결함을 가장 먼저 보완해야 하는지 위험도 및 시급성별로 취약점을 자동 덩어리 분류합니다.
2. **Selenium 기반 스크린 자동 캡처:** 인프라 트랙의 윈도우 서버(Windows Server) 환경 내에서 헤드리스(Headless) 웹 드라이버 가동 스크립트를 구현하여, 진단 자동화 탐색 과정 중 취약점이 매핑되는 순간의 증적 화면을 실시간으로 자동 캡처 및 자산화합니다.
3. **커스텀 한글 보안 진단 리포트 엔진:** 스캔된 통계 지표와 AI가 바인딩한 시큐어 코딩 대응 가이드를 결합하여, 가독성 높은 고품질의 표준 한글 보고서(PDF)를 원클릭으로 자동 생성 및 출력합니다.

# 4. 저장소 문서화 최신화 및 가상화 서버 스크립트 빌드 테스트 현황

실전 개발 및 플랫폼 기획 구체화와 동시에, 팀의 협업 표준화 유지를 위한 레포지토리 관리 및 인프라 사전 검증 작업을 수행했습니다.

- **백엔드(Backend) README.md 전면 수정 보완 (이예린):** 13일차에 전격 완료된 메인 통합 소스코드의 엔드포인트 구조와 최신 시스템 컴포넌트 사양을 백엔드 공식 기술 명세서에 완벽히 리비전 반영했습니다.
- **인프라(Infra) 사양 가이드라인 수정 반영 (박진아):** 클라우드 구성 토폴로지와 컨테이너 가상화 배포 사양을 인프라 공식 README 문서에 녹여내는 가이드라인 수정 작업을 활발히 진행 중입니다.
- **Selenium 가동 환경 스크립트 빌드 테스트 (유지수 / 박진아):** 가상화 서버 내에서 크롬 드라이버와의 호환성 충돌 없이 헤드리스 모드로 웹 페이지를 추적하고 스크린샷을 안정적으로 파일 시스템에 파일링할 수 있도록 사전 스크립트 빌드 및 구동성 테스트를 진행 중입니다.
- **보안 진단 역량 내재화:** 인프런(Infrun) 웹 보안 및 시큐어 코딩 대장정 전 교육 과정을 완강 달성하여, 플랫폼 내부 결함을 날카롭게 식별해 내기 위한 이론적 무장을 마쳤습니다.

# 5. Next Step: 진단 파이프라인 실제 코딩 및 크롤러 스크립트 실전 결합

이기종 보안 엔진을 제어할 시스템 아키텍처와 AI 매핑 규격이 완벽히 수립되었으므로, 차일 마일스톤부터는 설계 아트를 실제 소스코드로 형상화하는 본격적인 빌드 단계에 돌입합니다.

- **오픈소스 제어 파이프라인 구현:** 백엔드 서버 내에 ZAP API 호출 클라이언트 모듈 및 Semgrep CLI 결과 파일 파서 클래스 코드 작성 완료.
- **Selenium 크롤러 실전 결합:** 윈도우 서버 인프라 환경에서 검증 중인 Selenium 헤드리스 자동 캡처 스크립트를 플랫폼 백엔드 이벤트 핸들러와 물리적으로 연동.
- **AI 응답 데이터 파싱 모듈 테스트:** 프롬프트 매핑 템플릿을 통해 인입되는 LLM API의 JSON 응답 데이터를 자바 엔티티 객체로 변환하여 데이터베이스 무결성을 검증하는 통신 테스트 진행.
