# 1. 프로젝트 명 확정: Argus (아르고스)

팀원 의견을 모아 프로젝트 이름을 **Argus(아르고스)**로 최종 선정했습니다.

아르고스는 그리스 신화에 등장하는 **100개의 눈을 가진 거인**입니다. 잠들 때도 몇 개의 눈은 항상 뜨고 있어 철저하게 감시하는 존재처럼, **시스템의 보안 취약점을 상시 빈틈없이 스캔하고 찾아내겠다**는 의미를 담았습니다.

플랫폼의 핵심 뼈대는 **주요정보통신기반시설 가이드**와 **SK클라우드 가이드** 분석을 기반으로 잡았습니다. 이 표준 가이드라인을 기준으로 취약점 스캔 엔진의 기초 설계와 진단 로직을 구현하고, 이를 자동화 스캔 프로세스로 연결하는 방향을 조율했습니다.

# 2. 서브 도메인 분할 및 역할 분담 (Full-Stack & DB)

효율적인 병렬 개발을 위해 서비스 영역을 **5개의 서브 도메인**으로 나누고, 프론트엔드 UI부터 백엔드 비즈니스 로직까지 End-to-End로 책임지도록 역할을 나눴습니다.

## 풀스택 1 — 자산 및 권한 관리 (Asset & Auth Domain)

사용자 계정과 스캔 대상을 등록·검증하는 플랫폼의 **출입구**를 담당합니다.

- **Frontend (React):** 로그인/회원가입 UI, 타겟 자산(URL/IP) 등록 화면, 마이페이지(API 키 발급)
- **Backend (Spring Boot):** Spring Security 기반 JWT 인증, 회원/자산 CRUD API, **도메인 소유권 인증(DNS TXT / 파일 업로드 / 이메일)** 핵심 로직

## 풀스택 2 — 스캔 제어 및 모니터링 (Scan Pipeline Domain)

스캔 요청과 진행 상황을 실시간으로 확인하는 **조종석**을 담당합니다.

- **Frontend (React):** 스캔 시작/옵션 선택 화면, 진행률(프로그레스 바) 실시간 업데이트 UI
- **Backend (Spring Boot):** AWS SQS로 스캔 명령을 보내는 API, Polling/WebSocket 기반 상태 업데이트 API 및 DB 상태 반영

## 풀스택 3 — 취약점 가공 및 컴플라이언스 매핑 (Mapping Domain)

스캐너가 가져온 원시 데이터를 분석하고 가이드라인에 맞추는 플랫폼의 **두뇌**를 담당합니다.

- **Frontend (React):** 상세 진단 결과 리스트, 취약점별 KISA/SK 가이드라인 매핑 및 조치 안내 UI
- **Backend (Spring Boot):** 원시 데이터를 KISA/SK 기준과 조인해 가공하는 비즈니스 로직, 파싱 알고리즘 구현

## 풀스택 4 — 대시보드 시각화 및 리포트 엔진 (Reporting Domain)

분석 결과를 한눈에 볼 수 있게 정리하는 **결과물**을 담당합니다.

- **Frontend (React):** Chart.js 기반 통계 대시보드, 취약점 요약 UI, PDF 다운로드/뷰어
- **Backend (Spring Boot):** 취약점 통계 집계 API, **iText 기반 공식 PDF 리포트 렌더링** 로직

## 풀스택 5 — 이행 점검 및 사후 관리 (Remediation Domain)

조치 이후 재점검과 이력을 관리하는 **유지보수 사이클**을 담당합니다.

- **Frontend (React):** 조치 완료 상태 변경 UI, 재스캔 요청 화면, Before & After 비교 화면
- **Backend (Spring Boot):** 조치 이력 관리 API, 재스캔 데이터 병합/비교 로직, 이행 점검 리포트 데이터 추출 API

# 3. 인프라 트랙 역할 분담 및 아키텍처 설계

풀스택 영역을 받쳐줄 인프라와 보안 엔진 파이프라인도 역할을 나눴습니다.

## 인프라 1 — 클라우드 아키텍트 (AWS & CI/CD)

- 전체 AWS 인프라(`VPC`, `EC2`, `S3`, `RDS`) 세팅 및 네트워크 보안 관리
- React · Spring Boot 배포 환경 구성
- `GitHub Actions` 기반 자동 배포(CI/CD) 파이프라인 구축
- **서비스 흐름도 설계 완료**

## 인프라 2 — 보안 엔진 및 비동기 워커 (Python & SQS)

- `AWS SQS` 메시지 큐 구축
- `Python` 기반 백그라운드 스캐너 워커 프로세스 작성
- `Nuclei` / `Nmap` 오픈소스 스캐너 CLI 실행 및 결과 `JSON` 파싱 모듈 개발
