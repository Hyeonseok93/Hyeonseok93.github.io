# 서론

> **"전체 코드 통합과 플랫폼 거버넌스 확립을 마쳤습니다. 도메인 구입과 더불어 단순 툴이 아닌 유기적 '플랫폼'으로서의 온데(onde) 최종 점검을 완료했으며, Nginx 웹 서버 환경 정착, 결제 연동 리팩토링, 백엔드 전반의 헬스체크(Health Check) API 수립을 완수했습니다. 이와 동시에 인프런 웹 보안 대장정 강의를 완강하며 내일부터 전개될 실전 취약점 진단 및 수동 모의해킹을 위한 모든 전열을 가다듬었습니다."**
>
> main 브랜치 통합과 Nginx 도커 패키징, api/health 헬스체크 수립, 인프런 웹 보안 대장정 완강까지 마치며 실전 취약점 진단 페이즈를 준비했습니다.

# 1. 플랫폼 거버넌스 확립 및 인프라 실배포 패키징 (Nginx & Docker)

최종 런칭을 앞두고 서비스의 방향성 검증과 인프라 배포 자동화를 가속하기 위한 핵심 물리 계층 작업을 완료했습니다.

- **독립 도메인 구입 및 정체성 확정:** 서비스 배포 및 취약점 진단 타깃팅을 명확히 하기 위해 정식 도메인 구매를 완료했습니다. 팀 내부 회의를 통해 본 서비스를 단순한 개별 보안 진단 '툴(Tool)'에 가두지 않고, 예약 비즈니스 파이프라인과 보안 제어가 결합된 고도화된 **'통합 여행 플랫폼(Platform)'**으로 지속 고도화할 것을 최종 확정했습니다.
- **프론트엔드 Nginx 아키텍처 도입 및 도커라이징:**
  - 로컬 가상화 환경 및 상용 AWS 배포 단계에서 리버스 프록시 및 정적 에셋 라우팅을 안정적으로 받쳐줄 **Nginx 웹 서버 환경(`nginx.conf`)을 신설**했습니다.
  - 이를 프론트엔드 배포 가상 컨테이너 스펙에 이식하기 위해 **Dockerfile** 구조를 수정하여, 프로덕션 빌드 본이 Nginx 내부로 안전하게 탑재되어 오케스트레이션되도록 패키징을 완료했습니다.

# 2. 프론트엔드-백엔드 코어 파이프라인 통합 및 실전 리팩토링

개별 기능 브랜치로 분산되어 있던 대규모 연동 작업 소스코드를 공통 `main` 브랜치로 전면 병합 처리하며 플랫폼 최종 점검을 완수했습니다.

## ① 렌터카 위치 기반 기능(Location) E2E 정렬

- **백엔드 도메인 확장:** `Car` 핵심 엔티티에 위치 정보 지표인 `location` 필드를 신설하고, 검색 및 목록 조회 DTO, 레포지토리, 서비스 레이어를 전면 수정했습니다.
- **DB 마이그레이션 적용:** 변경된 스펙을 반영하기 위해 `V11__Add_Location_To_Rental_Cars.sql` 플라이웨이 스크립트를 빌드하여 실제 테이블에 컬럼 추가를 마쳤습니다. 프론트엔드 판매자 패널 및 검색 폼과 즉시 연동되도록 바인딩했습니다.

## ② 외부 결제 모듈(Portone) 제거 및 환경 변수화

- 아키텍처 정돈 및 구조 슬림화를 위해 기존에 적용되어 있던 Portone 외부 결제용 가이드 스크립트, 타입 정의 파일(`portone.d.ts`), 유틸 소스(`portOne.ts`) 등을 전면 자진 제거하고 관련 주석을 말끔히 청소했습니다.
- API 호출 URL의 모든 하드코딩 요소를 걷어내고 `.env.development` 및 `apiConfig.ts` 체계로 전격 환경 변수화하여 가시성을 보장했습니다.

# 3. 백오피스 권한(RBAC) 정밀화 및 가동 상태 모니터링 (Health Check) 수립

운영 거버넌스를 확립하고 마이크로서비스 가동 현황을 상시 감시하기 위한 보안 필터 및 모니터링 엔드포인트를 개설했습니다.

- **일반 관리자(USER_ADMIN) 권한 스펙 상향:** 기존에 총괄 관리자 등 상위 등급만 접근할 수 있었던 전사 대시보드 요약 지표 데이터(/summary) 및 통계 차트(/charts) API에 대해, 일반 운영 관리자인 **`USER_ADMIN` 역할도 유연하게 접근할 수 있도록 `@PreAuthorize` 보안 인가 사양을 확장**했습니다.
- **판매자 가입 제어 기능 구현:** 총괄 관리자가 악의적인 불법 판매자의 진입을 차단할 수 있도록, 어드민 유저 패널(`AdminUserPanel`)과 `adminApi` 내에 **판매자 회원가입 승인 및 거절 액션 메서드를 추가 고도화**했습니다.
- **전사 시스템 헬스체크(Health Check) 가동 (이예린):** 서버 컨테이너의 가동 무결성을 상시 모니터링하고 로드밸런서(ALB)와의 유기적인 타깃 그룹 체크를 지원하기 위해 백엔드 코어 및 어드민 내부 제어 계층 전반에 **`api/health` 헬스체크 모니터링 프로브 API를 신설** 및 구동 성공했습니다.

# 4. 프로젝트 기술 문서화 완수 및 인프런 보안 대장정 완강

- **통합 README.md 대폭 보강 (백엔드/프론트엔드):** AWS 클라우드 인프라 상세 아키텍처 스펙 구성 요소와 지속적 통합/배포(CI/CD) 자동화 파이프라인 구조를 설명하는 기술 명세 문서를 수백 줄 이상 보강했습니다. 또한 프론트엔드 `src/assets/readme/` 경로 내에 실제 구현 완료된 서비스 인터페이스 구동 화면 캡처본 18장을 자산화하여 매핑을 마쳤습니다.
- **웹 보안 진단 선행 학습 트랙 최종 수립:** SQL 인젝션 기초/심화 메커니즘 분석에 이어 OS 명령어 주입(OS Command Injection), XML 외부 엔티티 취약점(XXE), 크로스 사이트 스크립팅(XSS), 그리고 사이트 간 요청 위조(CSRF) 토큰 교차 검증 파일 업로드, 다운로드 취약점, 파라미터 변조 취약점, URL 접근 제한 미흡 취약점까지 연계되는 **인프런 웹 해킹/시큐어 코딩 전 교육 과정을 성공적으로 완강**했습니다. 이로써 플랫폼 내부 결함을 공격자의 눈으로 식별해 내기 위한 완벽한 지식 체계를 장착했습니다.

# 6. Next Step: 실전 컴플라이언스 웹 취약점 진단 및 모의해킹 가동

플랫폼 온데(onde)의 기능 개발 및 실서버 인터록 결합 마일스톤이 완벽하게 마감됨에 따라, 팀은 차일부턴 수동 취약점 분석 및 보안 진단 페이즈로 전격 전향합니다.

- **교차 검증 E2E 안정성 모니터링:** 렌터카 location API 연동 데이터의 무결성 체크 및 Portone 잔재 제거 이후의 전체 예약/결제 프로세스 수동 검증 진행.
- **Burp Suite 기반 실전 모의해킹 착수:** 인프런 선행 학습을 통해 도출한 웹 보안 취약점 시나리오를 바탕으로, 우리 시스템 내부의 파라미터 변조, 인증 우회, 데이터 강제 탈취(UNION/BLIND), 권한 불충분(BOLA) 결함을 발굴하기 위한 모의 스캔 및 공격 시뮬레이션 가동.
- **시큐어 코딩 리포트 작성 준비:** 도출된 취약점 결함에 대해 Prepared Statement 적용 상태 점검 및 다층 방어 구조 유효성을 검증하여 최종 조치 이행 리포트 빌드 예정.
