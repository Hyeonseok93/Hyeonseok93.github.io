# 1. 온데(Onde) 멘토링 피드백 반영

2차 멘토링에서 나온 공격 표면·비즈니스 로직 보완 요청을 온데에 반영했습니다.

## ① 커뮤니티 · 회원가입 · 판매자 대시보드

- **리뷰 댓글:** 비밀 댓글, 여행기 수정 기능을 추가했습니다.
- **회원가입:** 이름·전화·닉네임·나이 외에 **중복 확인**을 넣었습니다.
- **판매자 대시보드:** 매출이 어디서 나는지 집계해 보는 패널을 연동했습니다.

## ② 어드민 로그인 격리 · 다운로드 · 토스트

- **관리자 로그인:** 일반 로그인과 분리한 Admin 전용 로그인 화면을 만들었습니다.
- **다운로드:** 마이페이지 예약 내역에서 절대 경로를 쓰던 부분을 빼고 브라우저 다운로드로 바꿨습니다.
- **토스트:** 노출 시간을 3초 → 1초로 줄였습니다.

## ③ 마이페이지 · 정산 · 인프라 구성도

- **프로필 관리 (백엔드):** 사용자·판매자 각각 전화·닉네임·비밀번호·이메일 변경을 붙였습니다.
- **정산 상세:** 총액만 보이던 화면을 단건 예약 단위까지 보이게 바꿨습니다.
- **인프라 구성도:** 도메인 분리·윈도우 서버 결합에 맞춰 구성도를 업데이트했습니다.

# 2. 아르고스(Argus) 모듈 별 1인 개발 파이프라인

아르고스 쪽은 일정이 빠듯해서, 팀별로 엔진/Selenium/리포트를 나눠 만들던 구성을 접고 **모듈 별 1인 개발 파이프라인**으로 가져가기로 했습니다.

## ① A/B/C 분업 → 모듈 별 1인 파이프라인

- **기존:** A(엔진) / B(Selenium) / C(리포트)로 나누면 인터페이스 맞추는 데 시간이 많이 갈 것 같았습니다.
- **변경:** 공통 뼈대(Skeleton)만 같이 정해 두고, **한 사람이 스캔 엔진 → Selenium 증적 → 보고서까지** 이어서 만드는 모듈 별 1인 파이프라인으로 바꿨습니다. 엔진과 캡처가 같은 흐름 안에서 붙도록 맞춥니다.

## ② 진단 범위: DAST에 집중

- **SAST(Semgrep) 연동은 빼기:** 멘토링에서 들은 대로 오탐·실효 이슈가 커서 정적 분석 파이프라인은 제외했습니다.
- **DAST(ZAP) 중심:** 패킷·파라미터 주입과 응답 캡처 쪽에 맞추기로 했습니다.

## ③ 리포트 구조 · SK 가이드 매핑

- **보고서:** 사용자/판매자/관리자 3종 대신 **(일반·판매자 통합) 1종 + (본사 관리자) 1종**으로 줄였습니다.
- **SK 가이드:** Web/API 진단 가이드 8개 그룹 28항목 중 빠져 있던 매핑을 채워 넣었습니다.

# 3. Next Step

온데 피드백 반영과 아르고스 모듈 별 1인 파이프라인 방향을 잡은 뒤, 다음은 뼈대 확정과 온데 최초 스캔입니다.

- **뼈대 확정:** ZAP API ↔ Selenium 캡처 실행 순서·데이터 규격 정리

- **최초 보고서 초안:** 28항목 결과와 증적 스크린샷을 묶어 출력 테스트
