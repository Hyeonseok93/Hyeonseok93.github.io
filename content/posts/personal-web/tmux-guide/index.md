터미널 작업을 여러 창으로 나눠 관리할 때 **tmux**는 거의 필수에 가깝습니다. 이 글에서는 tmux의 핵심 개념과 자주 쓰는 단축키, 실전 팁을 정리합니다.

## tmux란 무엇이며 왜 쓰는가?

tmux(Terminal Multiplexer)는 하나의 터미널 세션 안에서 여러 작업 공간을 만들 수 있게 해 줍니다. SSH 연결이 끊겨도 세션이 유지되고, 개발·서버 작업을 훨씬 편하게 나눌 수 있습니다.

예를 들어 한 패널에서는 서버를 띄우고, 다른 패널에서는 로그를 보면서, 또 다른 윈도우에서는 에디터를 여는 식으로 구성할 수 있습니다.

## 세션(Session) 관리

세션은 tmux의 최상위 단위입니다. 프로젝트별로 세션을 나누면 컨텍스트 전환이 쉬워집니다.

```bash
tmux new -s dev
tmux attach -t dev
tmux ls
tmux kill-session -t dev
```

### 세션 이름 짓기 팁

프로젝트명이나 작업 목적을 세션 이름에 넣어 두면 `tmux ls`만 봐도 바로 구분할 수 있습니다.

## 윈도우(Window)와 패널(Pane) 단축키

기본 prefix는 `Ctrl+b`입니다. prefix 입력 후 아래 단축키를 사용합니다.

- `c` — 새 윈도우
- `%` — 세로 분할
- `"` — 가로 분할
- `방향키` — 패널 이동

## 스크롤 모드 및 실전 팁

로그가 길어지면 copy mode로 스크롤할 수 있습니다. `prefix + [`로 진입하고, `q`로 빠져나옵니다.

> tmux 설정은 `~/.tmux.conf`에 저장해 두면 매번 옵션을 다시 입력할 필요가 없습니다.

마우스 스크롤, 상태바 커스터마이징, 플러그인 매니저(tpm)까지 적용하면 개발 환경을 깔끔하게 정리할 수 있습니다.
